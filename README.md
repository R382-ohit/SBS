# Super Market Billing & Inventory System

A high-performance, command-line interface (CLI) Point-of-Sale (POS) and inventory management system built in C++. This project demonstrates Object-Oriented Programming (OOP) principles, efficient memory management, and persistent state storage.

## 🚀 Overview

This system allows administrators to manage supermarket stock and cashiers to process customer checkouts. It utilizes a highly optimized in-memory processing architecture, moving away from slow, disk-heavy operations to ensure lightning-fast item lookups and real-time stock validation. 

## ✨ Key Features

* **In-Memory Data Processing:** Utilizes C++ `std::unordered_map` (Hash Maps) to load the master inventory into RAM at startup, enabling **O(1) time complexity** for item lookups and updates.
* **Optimized Disk I/O:** Eliminates temporary file buffering and continuous disk reads/writes. Disk synchronization only occurs safely during startup and after bulk actions (like completing a checkout).
* **Dynamic Shopping Cart:** Employs `std::vector` to maintain an active memory state of a customer's cart, allowing for safe rollbacks if a transaction is canceled.
* **Case-Insensitive Search Algorithms:** Ensures smooth user experiences by successfully matching inventory records regardless of how the user capitalizes their input (e.g., matching "APPLE", "apple", and "Apple").
* **Robust Input Validation:** Prevents infinite loops, crashes, and corrupted data entry by blocking negative numbers and invalid string-to-integer inputs.

## 🛠️ Technical Stack

* **Language:** C++ (Standard C++11 or higher recommended)
* **Core Libraries:** `<iostream>`, `<fstream>`, `<unordered_map>`, `<vector>`, `<string>`, `<algorithm>`
* **Environment:** Windows OS (Utilizes `<windows.h>` for console clearing and sleep commands)

## 📂 System Architecture & File Structure

* `main.cpp` - Contains the core application logic, routing menus, and the `Bill` class.
* `Inventory.txt` - The persistent master database file automatically generated by the system. It stores items in the format: `[Item Name] : [Rate] : [Quantity]`.
